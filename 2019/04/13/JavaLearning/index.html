理解抽象、封装、继承和多态
封装中的访问控制修饰符
  • 四种访问级别的访问范围
    访问级别 | 访问控制修饰符 | 同类 | 同包 | 子类 | 不同包
    ——- | :———— | :— | :– | :— | :—-
    公开 | public | true | true | true | true
    受保护 | protected | true | true | true | false
    默认 | 没有修饰符 | true | true | false | false
    私有 | private | true | false | false | false
    某些想要访问到的却不可以直接访问的成员变量可以使用成员方法获取到
包-必要性
  1. 区分相同名字的类
  2. 当类很多时，可以很好的管理类
  3. 控制访问范围
  • 使用小写字母
继承:class 子类 extends 父类
  • 结论：public、protected、默认的修饰符所定义的属性和方法可以被子类继承
    privated修饰的属性和方法不可以被子类继承
重载(overload)
1.一个返回float类型的方法和一个返回int类型的方法，调用方法时参数中一个为int类型，一个为float类型，编译器会自动调用float类型的，因为是低精度的自动往高精度的转(float为高精度)
2.方法的重载是根据使用者给出的参数决定调用哪一个方法
  • 注意事项:
    1.方法名相同
2.参数的类型，个数，顺序至少有一个不同
3.返回类型不一样，不能构成重载
4.方法返回类型可以不同
5.方法修饰符可以不同
覆盖(override)
  • 注意事项:
    1.子类的方法的返回类型，参数，方法名称，要和父类方法的返回类型，参数，方法名称完全一样
    2.子类方法不能缩小父类方法的访问权限
多态
  • 一个引用(类型)在不同情况下的多种状态
  • 注意事项:类型的转换
    1.java允许父类的引用变量引用它的子类的实例(对象)
    Animal animal = new Cat();
    2.关于类型转换还有一些具体的细节要求…
抽象类
  • 注意事项:
    1.抽象类不能被实例化
    2.抽象类不一定要包含abstract方法.也就是说，抽象类可以没有abstract
    3.一旦类包含了abstract方法，则这个类必须声明为abstract
    4.抽象方法不能有主体.exp: abstract void aaa(){};不能有大括号
接口
  • 注意事项:
    1.接口不能实例化
    2.接口中所有方法都不能实现，不能有主体
    3.一个类可以实现多个接口
    4.接口中可以有变量(但变量不能用private和protected修饰)
  • 接口中的变量，本质上都是static的，不管加不加static修饰

  • 在java开发中，我们把经常用的变量，定义在接口中，作为全局变量使用.访问形式:接口名.变量名
    5.一个接口不能继承其它的类，但是可以继承别的接口

  • 接口就是给出一些没有内容的方法，封装在一起，到某个类要使用的时候，再根据具体情况把这些方法写出来
    小结:接口就是更加抽象的抽象的类，抽象类里的方法可以有方法体，接口里的方法都没有方法体.
    接口体现了程序设计的多态和高内聚低耦合的设计思想

接口实现多态
  • 前期绑定:在编译的时候就已经确定是哪一种类型
  • 后期绑定:在运行的时候才知道是哪一种类型
final
  • 可以修饰变量和方法
    1.不希望父类的某个方法被子类覆盖(override)时,可以使用final
    2.不希望父类中某个变量被修改时，可以用final，父类中定义的变量尽量给其赋初值，若不赋初值，可能指向一个空地址，换言之，程序可能不结实，若用final修饰，必须赋初值,final修饰的变量叫常量,用xx_xx_xx来命名
    3.不希望类被继承时，可以用final修饰

ArrayList
  • 字符串用“==”比较，是比较地址是否相等，比较值是否相等，需使用equals
HashMap和Hashtable的区别
  • 相同点:都是java的集合类，都可以用来存放java对象
  • 不同点:
    1.历史原因：Hashtable是基于Dictionary类的，HashMap是Java 1.2引进的Map接口的一个实现
    2.同步性:Hashtable是同步的,这个类中的一些方法保证了Hashtable中的对象是线程安全的.而HashMap是异步的,因此HashMap中的对象并不是线程安全的.因为同步的要求会影响执行的效率,所以若果不需要线程安全的集合那么使用HashMap是一个很好的选择,这样可以避免由于同步带来的不必要的性能开销,从而提高效率.
    3.值:HashMap可以让你将空值作为一个表的条目的key或value,但Hashtable是不能放入空值的(null)
Vector和ArrayList的异同点
  • 相同点:都是java的集合类，都可以存放java对象
  • 不同点:
    1.Vector是同步，所以安全，ArrayList是异步的，所以不安全，但效率高
2.Vector的数据增长是增长其自身的一倍，ArrayList是增长其自身的50%，若存放大量数据，需用Vector，可以通过设置集合的初始化大小避免资源的消耗
集合框架小结

1.若要求线程安全,使用Vector、Hashtable
2.若不要求线程安全,应使用ArrayList,LinkedList,HashMap
3.若要求键值对,则使用HashMap,Hashtable
4.若数据量很大,又要线程安全考虑Vector

泛型
  • 定义一个类，但是类的具体类型不确定，可以用泛型定义，根据具体情况确定类
  • 泛型的本质是参数化类型，也就是所操纵的数据类型被指定为一个参数.这种参数类型可以用在类，接口和方法的创建中,分别被称为泛型类、泛型接口、泛型方法.
  • 泛型的好处就是安全简单，在编译的时候检查类型安全，并且所有的强制类型转换都是自动和隐式的，提高代码的重用率
  • 优点:
    1.类型安全
2.向后兼容
3.层次清晰
4.性能较高，用GJ编写的代码可以为java编译器和虚拟机带来更多的类型信息，这些信息对java程序做进一步的优化提供条件

异常处理

  • 2种方法处理:
    1.在发生异常的地方直接处理
    2.将异常抛给调用者,让调用者处理

  • 异常分类:
    1.检查性异常，编译器事先能确定的异常
    2.运行期异常
    3.错误:可能源于bug,更可能源于环境问题

图形用户界面(swing)
  • 布局管理器
    1.组件在容器中的位置和大小由布局管理器决定,所有的容器都会使用一个布局管理器,通过它来自动进行组件的布局管理
    2.种类有五种: 流式布局管理器(FlowLayout)、边界布局管理器(BorderLayout)、网格布局管理器(GridLayout)、卡片布局管理器(CardLayout)、网格包布局管理器(GridBagLayout).前三种是常见的布局管理器
    1.边界布局管理器(BorderLayout)
    注意事项:
    + 不是五个部分都必须添加
+ 中部组件会自动调节大小
+ JFrame，JDialog默认布局管理器就是BorderLayout
    2.流式布局
    注意事项:
    + 不限制他所管理的组件大小,允许他们有最佳大小
+ 当容器被缩放时,组件的位置可能有变化，但组件的大小不变
+ 默认组件是居中对齐，可以通过FlowLayout(int align)函数来指定对齐方式
